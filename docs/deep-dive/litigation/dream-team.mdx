// Conspiracy Network Visualization Query
MATCH path = (defendant:Person)-[:CONSPIRES_WITH*1..3]-(coconspirator:Person)
WHERE defendant.role_in_conspiracy = 'Primary Defendant'
WITH path, nodes(path) as conspirators, relationships(path) as connections
RETURN path, conspirators, connections
ORDER BY length(path) DESC
LIMIT 100

MATCH (pa:Predicate_Act)-[:SUPPORTS]->(claim:RICO_Claim)

RETURN pa, actor, doc, claim

## BENCH_AGENT: Automated Expert Coordination

### Purpose

Coordinate with Federal Litigation Dream Team Database for case-specific expert matching.

### Workflow

```yaml
BENCH_AGENT Workflow:
  1. Analyze case scope + expert needs
  2. Query dream team database for matching profiles
  3. Match case requirements → expert credentials
  4. Generate recommended expert roster
  5. Track expert availability + engagement status
```

### Implementation

```python
class BenchAgent:
    """
    Dream team coordination agent powered by Vertex AI.
    """
    
    def __init__(self, database: DreamTeamDatabase):
        self.database = database
        self.matcher = ExpertMatcher()
        self.tracker = EngagementTracker()
    
    async def recommend_experts(self, case_scope: CaseScope) -> ExpertRoster:
        # Query dream team database
        candidates = await self.database.query(case_scope.requirements)
        
        # Score and rank candidates
        scored_candidates = await self.matcher.score(candidates, case_scope)
        
        # Generate tiered roster
        roster = ExpertRoster(
            tier_1_lead_counsel=self._filter_tier(scored_candidates, tier=1),
            tier_2_specialists=self._filter_tier(scored_candidates, tier=2),
            tier_3_expert_witnesses=self._filter_tier(scored_candidates, tier=3),
            tier_4_academics=self._filter_tier(scored_candidates, tier=4),
            tier_5_technical=self._filter_tier(scored_candidates, tier=5)
        )
        
        return roster
```
